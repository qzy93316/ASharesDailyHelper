# 消息面分析(News Analysis)

日报的技术面由代码算,**消息面靠联网搜索 + 归纳**。本 skill 把这套工作流固化下来:读当日荐股池 → 定向联网搜当天政策/热点/公告 → 结构化归纳 → 与技术面交叉 → 写回报告。对应设计文档 §3.3 消息面引擎、决策层"技术60%+消息30%+大盘10%"。

## 何时触发

用户说「今日热点 / 消息面 / 补消息面 / 查政策 / 今天有啥利好利空 / 热点分析」,或刚生成日报想补齐第三章。

## 执行步骤(我按此顺序做)

1. **读当日结构**:打开 `reports/日报-YYYY-MM-DD.json`,取出 `sectors`(强势板块)与 `picks`(荐股代码/名称/板块)。没有侧车就读 `.md`。
2. **定向联网搜索**(WebSearch/WebFetch,注意用当前月份关键词):
   - 大盘/政策:`当日 A股 收盘 政策 消息面 <当月>`、`央行/发改委/行业政策 <当月>`
   - 每个强势板块:`<板块名> 板块 大涨 原因 催化 <当日>`
   - 重点个股:`<股票名> 公告 消息 <当月>`(尤其查有无利空:解禁/减持/业绩预警)
3. **结构化归纳**(每个板块一段,固定四要素):
   - **板块** → **催化剂是什么**(政策/事件/资金/传闻)→ **是短期情绪还是中期逻辑** → **受益个股**(标注是否在今日荐股池)
4. **与技术面交叉**(决策纪律):
   - 只有「消息面有催化 **且** 技术面评分≥60」的板块才重点推;
   - 荐股池里若某只**技术好但消息面查到利空**(解禁/减持/监管),必须标红提示、下调优先级。
5. **写回报告**:把归纳好的 Markdown 存成 `news.md`,执行注入:
   ```bash
   python skills/news-analysis/scripts/inject_news.py --date YYYY-MM-DD --news news.md
   ```
   之后可再跑 render-html 出图文版。

## 输出模板

```markdown
### 大盘与政策
<一句话概括当日大盘情绪 + 关键政策/宏观事件>

### <板块A>(技术面评分区间)
- **催化剂**:<政策/事件/资金,附来源>
- **性质**:短期情绪 / 中期逻辑(说明理由)
- **受益个股**:<名称(代码),标注★=在今日荐股池>
- **交叉结论**:<消息面+技术面是否共振>

### 风险提示
<查到的利空:解禁/减持/业绩雷/监管;若无则写"未见明显利空">
```

## 纪律与边界

- **务必标注来源**,区分「已证实(公告/官媒)」与「传闻/小作文」,后者明确标注不可尽信
- **主动找利空**,不只报喜(呼应"强制反方理由");查到解禁/减持/问询函必须写进风险提示
- 搜索结果稀疏或时效不足时,**如实说明"当日公开消息有限"**,不编造催化剂
- 不预测涨跌、不承诺;消息面仅供与技术面交叉参考
- 与 [backtest-review](../backtest-review/SKILL.md) 配合:复盘时可回看"当初的消息面逻辑是否兑现"
