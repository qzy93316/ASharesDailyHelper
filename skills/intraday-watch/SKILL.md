# 盘中监控(Intraday Watch)

日报是日频快照,盘中价格到了计划价位时没人提醒。本模块独立循环,批量快照盯 持仓+荐股池,**只在触及作战方案里事先算好的计划价位时**桌面弹窗——系统帮守纪律,不做盘中临时喊单。

## 🔒 边界

- 只读行情、只提醒,**不接触账户密码、不代下单**。
- 告警**只绑事先写好的计划价位**(来自作战方案 `judge` 价位),不追盘中临时信号。
- 免费源盘中只盯少量个股(≤15);全市场扫描仍在日频/盘前。

## 何时触发

用户说「盯盘 / 盘中监控 / 开始监控 / 到价提醒 / 看着我的票 / watch」。**须在用户本机、交易时段运行**(沙箱休市/限流跑不了实时)。

## 前置

当日先有作战方案:`python scripts/run_workflow.py`(或 `action_plan.py`)→ 生成 `作战方案-YYYY-MM-DD.json`。watch 从中提取监控清单与计划价位。

## 运行

```bash
python scripts/watch.py                      # 读当日作战方案,循环监控(交易时段)
python scripts/watch.py --once               # 单轮冒烟测(休市也能测阈值判断)
python scripts/watch.py --interval 20 --date 2026-07-05
```

- 轮询间隔、每类告警上限、触发缓冲、交易时段限制 见 `config.yaml` 的 `watch:` 段
- **桌面弹窗**:优先 `win11toast`(建议 `pip install win11toast`,Win10/11 原生 toast);无则退 `win10toast`;再无则 `MessageBox` 兜底(守护线程非阻塞)+ 蜂鸣
- 产出:控制台实时打印 + `reports/YYYYMMDD/盘中告警.log`(告警台账)+ `盘中监控-分时.html`(**自刷新分时页**,双击即开,ECharts 分时线 + **成交量柱** + 计划价位标线,每20秒自刷新)+ `分时缓存.json`(全天分时数据缓存)

## 分时图(价 + 量,全天完整)

- **数据源**:分时走腾讯全天分时接口 `ifzq.gtimg.cn`(与告警用的批量快照 `qt.gtimg.cn` 分离),每分钟拉一次,返回当日 **9:30→当前**整段的每分钟价 + 累计成交量。
- **不依赖启动时间**:因为拿的是全天段而非自启动起攒点,所以**几点启动、中途重启都不缺前半段**。每分钟数据落地 `reports/YYYYMMDD/分时缓存.json`(每只约150行、单日几十KB),开机先读缓存、拉取失败自动沿用磁盘缓存;收盘后/周末打开分时页仍是完整全天图。`--once` 单轮即拉全天。
- **成交量柱**:分时页下方红绿量柱(手),累计量差分成每分钟量、随分钟涨跌红绿上色(涨红跌绿),与现价 tooltip 联动——放量突破/缩量回踩一眼可辨,短线择时的关键辅助。

## 告警类型(全部绑计划价位)

- ⛔止损:跌破结构止损
- 🎯介入:回踩到建仓/逢低吸价(下行到位)
- 🚀突破:突破压力/逢高建仓价(上行到位)
- 🔺反弹减:反弹到减仓触发价(上行到位)
- 📈涨停 / 💥炸板(涨停后开板):用涨停价现算
- 去重:每股每类每日限 N 次(config `max_alerts_per_type`)

## 口头研判

弹窗只是"到价提醒",到价后仍由用户决定观望还是动手。被问到时结合作战方案给"到这个价位该怎么应对"的研究性建议(守/减/介入),强调非喊单、请自行复核。
