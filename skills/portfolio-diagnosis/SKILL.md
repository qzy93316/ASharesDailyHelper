# 持仓诊断(Portfolio Diagnosis)

把用户**真实持仓**接入分析引擎做深度体检:逐只结合**成本价**给"持有/减仓/止损/观察"的规则化诊断,并从组合层面看仓位、集中度、主力资金、主线暴露。对应设计文档二期「持仓诊断」。

## 🔒 安全边界(必须遵守)

- **绝不接收、索取、使用证券账户密码/交易密码/资金密码**。用户若提出提供账号密码,礼貌但坚决拒绝,并说明只需导出的持仓文件即可。
- **只读诊断,绝不代下单/代交易**。本 skill 不连接任何交易接口。
- 持仓文件从券商/同花顺 APP **手动导出**(无需任何凭证),放入 `portfolio/`。

## 何时触发

用户说「诊断持仓 / 看我的持仓 / 持仓体检 / 分析我的股票 / 持仓分析」,或刚把持仓文件放进 `portfolio/`。

## 执行步骤

1. **确认持仓文件**:看 `portfolio/` 下有无持仓文件(`.xls`/`.xlsx`/`.csv`)。没有就提示用户从券商/同花顺 APP 导出持仓表放进去(就是那张 代码/名称/数量/成本价/市价 的表)。
2. **跑诊断**(在用户机器上,联网取行情):
   ```bash
   python scripts/diagnose_portfolio.py            # 自动取 portfolio/ 最新文件
   python scripts/diagnose_portfolio.py 持仓.xls   # 或指定文件
   ```
   脚本兼容同花顺导出的 GBK 制表符"xls"、通用 csv、真 xlsx;自动识别中文列名。
3. **产出**:`reports/YYYYMMDD/持仓诊断-YYYY-MM-DD.md`(+ `.json` 侧车)。逐只给:
   - 成本 vs 现价、盈亏%、信号/评分、**结构止损**、成本感知**诊断标签**(止损/重亏警戒/逢反弹减/持有/持有观察/观察)
   - 组合体检:市值/浮盈亏/亏损家数、仓位%、最大单票集中度、主力净流出家数
4. **(可选)出交互图表版**:
   ```bash
   python skills/render-html/scripts/report_to_html.py "reports/YYYYMMDD/持仓诊断-YYYY-MM-DD.json"
   ```
5. **口头研判**:基于诊断结果给组合层面建议(减/守/换),强调"研究参考非喊单",并主动给反方理由与风险。
6. **(可选)与 [trading-memory](../trading-memory/SKILL.md) 打通**:诊断出的持仓可记入操盘台账的未平仓,复盘时对照。

## 诊断规则(成本感知,规则化透明)

按优先级(全部由 `diagnose_portfolio.py` 代码判定,AI 只解读):

- **已跌破结构止损** → `止损`:纪律应减仓/离场,勿扛单
- **浮亏超止损容忍(config `max_drawdown_pct`)约1.5倍** → `重亏警戒`:不补仓摊低,反弹减亏
- **趋势偏弱 + 主力近5日净流出** → `逢反弹减`
- **结构健康 + 资金未流出** → `持有`
- **趋势偏弱但主力逆势吸筹** → `持有观察`
- 其余 → `观察`(守结构止损)

## 关于"自动同步实盘账户"

用户可能希望免去手动导出、自动拉取实盘持仓。技术上可行但**必须**:①在用户自己电脑上跑;②走券商官方量化接口(如 QMT/miniQMT)的官方授权,**密码只存在官方客户端、绝不经手 AI/第三方**;③**仅只读持仓,绝不自动下单**。性价比通常不如"手动导出 CSV(5秒)",除非用户强需求。默认推荐 CSV 导入。

## 能力边界

- 只做"读持仓 + 技术/资金/基本面诊断"的研究参考,不预测涨跌、不承诺、不下单
- 行情取自免费接口,可能延迟;基本面为披露口径;止损为结构化估算
- 持仓文件为个人敏感数据,`portfolio/` 已在 `.gitignore` 中,不会被提交/开源
