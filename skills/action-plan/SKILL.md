# 每日作战方案(Action Plan)

把散落在各股 judge 里的"介入价/止损/盈亏比"和持仓诊断的"减/守/清"标签,**跨股合成为一份整体作战方案**:今天/明天整体怎么操作,统筹现有持仓与荐股池,每个动作都绑**事先算好的计划价位**。对应对话里那种"现仓加减、新标建仓"的组合研判。

## 🔒 边界

- 研究性组合研判,**非实时喊单、不代下单、不接触账户密码**。
- 所有动作绑**事先算好的计划价位**(判据来自代码 `judge.py`/持仓诊断),系统帮守纪律不帮破。

## 何时触发

用户说「作战方案 / 今日怎么操作 / 给我操作建议 / 加减仓建议 / 持仓和荐股怎么搭 / 组合方案」。

## 执行

前置:当日已有 `日报-*.json`(荐股池,跑 `run_workflow`/`daily_report`)、可选 `全局池-*.json`、`持仓诊断-*.json`(有持仓文件时 `run_workflow` 会自动诊断)。

```bash
python scripts/action_plan.py --date YYYY-MM-DD          # 持仓+荐股池全量
python scripts/action_plan.py --focus 持仓               # 只看持仓
python scripts/action_plan.py --focus 002171,冰轮        # 只看指定代码/名称(用户可用提示词收窄)
```

产出 `reports/YYYYMMDD/作战方案-YYYY-MM-DD.(md/json)`。也可 `run_workflow` 一条龙(两池→诊断→作战方案→渲染,顶部出「🎯今日作战方案」卡片)。

## 输出内容

- **组合姿态**:大盘 regime + 情绪周期 → 总仓位建议 / 是否可新建仓 / 单票集中度预警
- **持仓端**:每只 → `清仓/止损` `减仓` `持有` `逢低加仓` + 触发价 + 幅度 + 依据
- **荐股端**:每只 → `建仓` `逢低吸` `逢高突破建仓` `观察` `放弃` + 计划价/止损/目标/盈亏比
- **换股**:卖弱(持仓趋势资金双弱)→ 买强(荐股 quality 高)

## 与其它模块

- 计划价位是下一步 **盘中监控 `watch.py`** 的告警阈值来源(触及即桌面弹窗)
- 与 [portfolio-diagnosis](../portfolio-diagnosis/SKILL.md)(持仓端输入)、[news-analysis](../news-analysis/SKILL.md)(消息面交叉)、[backtest-review](../backtest-review/SKILL.md)(事后验证方案质量)配合

## 口头研判要点

基于作战方案给用户说人话的组合建议:先说组合姿态(仓位几成、能不能加),再说该减的持仓、该建的新股、换股对;强调计划价位与失效条件;主动给反方理由。**研究参考,非投资建议。**
