# 盘后/周度复盘验证(Backtest Review)

用荐股池当初给出的**计划(目标价/止损价)**对照之后的**真实K线走向**,客观判定每只荐股的成败,并把结果累积成**历史成功率**——让系统的推荐有据可查、可迭代。这是设计文档「盘后验证总结 / 正确性保障 §6」的落地。

## 何时触发

- 盘后:"复盘一下今天的荐股""今天推的几只走得怎么样"
- 周末:"这周荐股表现""本周复盘"
- 任何时候:"系统荐股的历史命中率""回测一下荐股池"

## 何为一次复盘:检验"上一交易日决策"的对错

复盘的核心是**用今日结果检验上一交易日日报里各项荐股决策的对错**,不出作战方案
(作战方案留给次日盘前工作流结合最新消息面再出)。归档/复盘统一锚定"最近一个交易日的操作版报告"。

**三池合验**:热点池(`日报-*.json`)+ 全局池(`全局池-*.json`)+ 盘中机会池(`盘中机会池-*.json`),
各自打 `pool` 标签,`stats` 出 **by_pool** 分池命中率(看哪个池的决策更靠谱)。盘中池 pick 只有
`close/soft_stop`,脚本自动归一(目标按 `target_gain_pct` 反推)。

**荐股维度 vs 持仓维度(必须隔离)**:本复盘只检验**荐股当初的计划**(建仓价/目标/止损)对不对,
与你的实际持仓成本**无关**;持仓健康度是**另一维度**,由 `diagnose_portfolio` 按你的成本单独诊断。
荐股池往往不会立即买入,即使买了也从两个维度分开看,不可混。

## 判定口径(借鉴 a-share-paper-trading 成交模型)

对每只荐股,取其荐股日之后的 N 个交易日(**默认 1 = 只看次日**;检验"昨日决策今日对错"的口径。
`--hold` 可放大到波段视角),逐日判定:

1. **先触目标价** → ✅ 达标(胜)
2. **先触止损价** → ❌ 止损(败)
3. 都没碰(或无计划价位)→ 看期末收盘:收正为「持有小胜」,收负为「持有小负」

> 同日既触目标又触止损时,保守判为「先触止损」(不给系统贴金)。
> 另记录:次日涨幅、持有期涨幅、最大浮盈、最大浮亏,供研判风险收益比。

数据来自主项目 `scripts/fetcher.py` 的韧性数据层(缓存+双源),复盘用的后续K线即其近端历史。

## 用法

**用户说「复盘/盘后复盘」→ 一键脚本 `scripts/run_review.py`**(荐股维度 + 持仓维度**同时触发**,
无需再单独说「诊断持仓」),它依次跑:① `review.py daily`(荐股维度三池)→ ② `diagnose_portfolio.py`
(持仓维度,有 portfolio/ 持仓文件才跑)→ ③ `review_to_html.py` 合成**暗色图文 HTML**(复用盘前报告
主题、宽表;两维度分两模块展示、互不污染)。作战方案**不在复盘出**(留给次日盘前工作流)。

```bash
# 一键盘后复盘(荐股维度 + 持仓维度 → 合成 HTML)—— 推荐
python scripts/run_review.py 2026-07-06            # 指定日期;不带则今天
python scripts/run_review.py 2026-07-06 --hold 5   # 波段视角

# 分步(需要时):仅荐股维度
python skills/backtest-review/scripts/review.py daily --date 2026-07-06

# 波段视角(看 5 日)
python skills/backtest-review/scripts/review.py daily --date 2026-07-06 --hold 5

# 一周区间复盘(周末汇总)
python skills/backtest-review/scripts/review.py range --since 2026-06-30 --until 2026-07-04

# 累积历史成功率(总/按池/按信号/按评分档/按策略/按情绪阶段)
python skills/backtest-review/scripts/review.py stats
```

- 复盘结果写到 `reviews/复盘-*.md` + 侧车 `reviews/复盘-*.json`;`run_review.py` 自动用
  `review_to_html.py` 合成暗色图文 HTML(荐股维度 + 持仓维度两模块)
- 每次验证追加到累积台账 `reviews/review_ledger.jsonl`(按 日期+代码 去重),`stats` 据此算长期命中率
- **依赖侧车**:复盘读取 `reports/日报-YYYY-MM-DD.json`(由 daily_report 生成);当天荐股需等之后有交易日才可验证,否则标「待验证」

## 解读要点(AI 拿到结果后怎么说)

- **命中率单看不够**:结合平均最大浮盈/浮亏看**风险收益比**;命中率 50% 但盈亏比 3:1 依然是好策略
- **按评分档看单调性**:理想情况「≥75」命中率应高于「60-74」,否则说明评分体系需校准
- **达标 vs 止损结构**:止损占比过高 = 追高/选时问题;持有小负多 = 目标价定太远
- 用累积样本(`stats`)而非单日结论下判断;样本少时明确说明"样本不足,仅供参考"
- 复盘是为了**迭代 config.yaml 的门槛与风格**,不是自我安慰

## 能力边界

- 只做事后验证与统计,不预测、不承诺;判定基于免费日线数据,不含盘中细节与滑点
- 与 [trading-memory](../trading-memory/SKILL.md) 互补:此技能验证「系统荐股」的有效性,操盘记忆记录「你的真实操作」的盈亏
- 复盘 MD 可交给 [render-html](../render-html/SKILL.md) 转成图文版归档
