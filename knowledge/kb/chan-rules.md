## 0. 总纲:走势的分解、递归与结合律

**【定义】(原书口径)**
- 任何级别的走势,都可以唯一地分解为「走势类型」的连接。走势类型只有三种:**上涨、下跌、盘整**。
- 递归关系:**低级别的若干走势类型,连接成高级别的一个走势类型**。分解链条为:
  `K线 → (包含处理) → 分型 → 笔 → 线段 → (线段构成)中枢/走势类型 → 更高级别中枢/走势类型 …`
- 结合律:相邻同向走势类型可结合为更大级别的一段;级别是**相对**的(由中枢级别决定),不是固定周期。
- 核心命题:**走势终完美**——任何级别的一个走势类型走完,必然进入下一个走势类型(方向不预设)。
- **「没有趋势,没有背驰」**:背驰只在趋势(至少两个同向中枢)里讨论;盘整只有盘整背驰。

**【可执行判定】**
1. 一切判定建立在**已去包含的合并K线序列**上,而非原始K线。
2. 引擎自底向上逐层构建:`bars → merged → fractals → bi → seg → zhongshu → 走势类型`。
3. 「级别」在代码中用**周期档位**近似(如 5m/30m/日/周);同一档位内用「中枢个数与嵌套」来表达相对级别。
4. 任何买卖点/背驰判断都必须**指明级别**(在哪个周期序列上算的),否则无意义。

**【对我们引擎的建议】**
- 现有 `analyze()` 是单层(单周期)流水线,缺「线段」和「级别递归」两层。建议:
  - 先补齐**线段层**(见 §5),中枢改为**建立在线段/次级别走势之上**而非直接建立在笔上(见 §6)。
  - 引入**多周期入口**:对同一标的传入 {30m, 日, 周} 三档 bars,分别 `analyze`,再做跨级别对齐(见对接清单)。

## 1. K线包含关系处理

**【定义】(p153/p154)**
- 两K线若一根的高低完全被另一根包含(高≤高且低≥低,或反之),即互相包含,需合并为一根。
- 合并方向由**当前走向**决定:
  - 向上处理:取两者**高点的高**、**低点的高**(高高、低取高)。
  - 向下处理:取两者**低点的低**、**高点的低**(低低、高取低)。
- 合并后的K线继承后一根的时间戳;第一二根的方向由二者高点比较确定。

**【可执行判定】**(等价于现有 `_merge_inclusion`)
```
direction = 由前两根非包含K的高点比较得到(高→up,低→down)
遇到与前一合并K互含时,按 direction 用 高高/低取高 或 低低/高取低 合并
不含时,更新 direction,推入新合并K
```

**【对我们引擎的建议】**
- 现有实现基本正确,但 `direction` 初始化写死为 1(向上),遇到开头就下跌的序列会误判首合并方向。
  建议:用**前两根原始K的高点比较**来初始化 `direction`,而不是常数 1。
- 合并K应保留 `o/c` 或至少保留原始 index 区间(供后续线段"缺口"判断用真实价差)。

## 2. 分型(顶分型 / 底分型)

**【定义】(p153)**
- 在**去包含后**的序列上,连续三根K线 `k1,k2,k3`:
  - **顶分型**:`k2` 的高点是三者最高,且 `k2` 的低点也高于 `k1、k3` 的低点。
  - **底分型**:`k2` 的低点是三者最低,且 `k2` 的高点也低于 `k1、k3` 的高点。
- 顶分型的**顶点**=k2 高点;底分型的**底点**=k2 低点。分型是笔的端点候选。

**【可执行判定】**(现有 `_fractals` 只比高/只比低,可加严)
```
top:  m[i].h > m[i-1].h and m[i].h > m[i+1].h  且  m[i].l > m[i-1].l and m[i].l > m[i+1].l
bot:  m[i].l < m[i-1].l and m[i].l < m[i+1].l  且  m[i].h < m[i-1].h and m[i].h < m[i+1].h
```

**【对我们引擎的建议】**
- 现有 `_fractals` 只判高点/低点单边,未加"另一端也顺序"的约束。因已在去包含序列上,漏判概率低,
  但建议补上双边条件以贴合原书,减少震荡区噪声分型。

## 3. 笔(新笔 / 老笔)

**【定义】(p153/p154)**
- 一笔 = 一个顶分型到相邻底分型(或反向)之间的连接,方向明确。
- 成笔条件(原书两个口径,项目应固定其一):
  - **老笔(严)**:顶分型与底分型之间,**除共用K外至少还有 1 根K**,即两分型的 k2 之间(含端点)**至少 5 根去包含K**;且顶底不共用K。
  - **新笔(宽)**:两分型之间(去包含K)**独立K数≥1**,即顶底 k2 间隔 `≥ 3 根`(顶 k2、中间≥1、底 k2)。
- 相邻笔必须**顶底交替**;若连续出现同类分型,取更极端者(更高的顶/更低的底)。

**【可执行判定】**(现有 `_bi`,`min_gap=3`)
```
交替连接顶/底分型;同类相邻取更极端者;
异类相邻需 f.i - last.i >= min_gap(去包含K索引间隔)才成笔
```

**【对我们引擎的建议】**
- 现有 `_bi` 的 `else` 分支里"间隔不足"时逻辑不完整(有个 `pass` 空分支),会**丢弃间隔不足的分型而不做任何替换**,
  可能漏掉更极端的端点。建议:间隔不足且比"上上个同类端点"更极端时,应回退合并,而不是直接丢。
- 把 `min_gap` 做成参数,新笔=3、老笔=5,由调用方按需选择。

## 4. 线段(尚未实现,重点补充)

**【定义】(p155/p156/p157/p158)**
- 线段由**至少 3 笔**构成,且这 3 笔必须**有重叠**(否则不成线段)。方向与首笔一致。
- **特征序列**:向上线段取所有**向下笔**、向下线段取所有**向上笔**,把每笔抽象为一个 `(高,低)` 元素,按时间排成序列。
- **特征序列的分型**决定线段是否结束:
  - 在特征序列元素之间同样做**包含处理**,然后找特征序列的顶/底分型。
  - 线段被"破坏/终结"于特征序列出现**反向分型**。
- **缺口(第一种/第二种情况)**:
  - **无缺口(第一种)**:特征序列相邻元素**没有价格缺口**(前一元素与后一元素重叠),则出现一个标准反向分型即确认线段结束。
  - **有缺口(第二种)**:特征序列在分型处出现**缺口**(元素间不重叠),则该分型**不立即确认**;需要在缺口之后
    的反向特征序列里**再出现一个分型**(即反向线段自身要能成立)来确认原线段结束,否则视为原线段延续。

**【可执行判定】**
```
输入:已完成的 bi 序列(带方向、端点价、index)
1. 确定候选线段方向 dir = 首笔方向
2. 取反向笔构成特征序列 seq_i = [(h,l) of 每根反向笔]
3. 对 seq_i 做包含处理(同 §1,方向随线段方向)
4. 在处理后的特征序列上找分型(同 §2)
5. 若该分型处 前后元素重叠(无缺口):确认线段在此结束,起一新反向线段
   若有缺口:标记"疑似结束",等待缺口后的反向特征序列也形成分型 → 再确认;
             否则原线段延续,合并该波动
6. 线段端点 = 对应笔的端点极值
```

**【对我们引擎的建议】**
- **这是当前最大的缺口**:`chan.py` 直接用「笔」建中枢,跳过了线段。缠论标准中枢是**建立在次级别走势(≈线段)**上的,
  用笔建中枢会产生大量过碎、不稳定的中枢。建议新增 `_segments(bi)`,输出 `[{d_start,d_end,dir,high,low}]`。
- 缺口判定需要**真实价格连续性**,因此特征序列元素要携带对应笔的真实高低价(用去包含K的极值)。

## 5. 中枢(次级别三段重叠)

**【定义】(p072/p073/p074)**
- 中枢 = **某级别走势中,至少三个连续的次级别走势类型(≈三段线段/次级别走势)有价格重叠区间**。
- **中枢区间**:
  - `ZG`(中枢上沿)= 前三段中,各段**高点的最小值**(即三段高点的最低者)。
  - `ZD`(中枢下沿)= 前三段中,各段**低点的最大值**(三段低点的最高者)。
  - 成立条件:`ZG > ZD`(三段确有公共重叠区)。
- **中枢延伸(震荡)**:后续走势的高低点仍落在 `[ZD, ZG]` 附近来回,则中枢**延伸**,中枢区间不变(可用第一次三段确定后锁定)。
- **中枢扩展/扩张**:当出现**更高级别**的重叠(多个同级别中枢由高一级中枢连接),级别升一级。
- **中枢新生**:走势离开原中枢(带背驰或第三类买卖点)后,在新的价格区域由新的三段重叠形成新中枢。
- 中枢的级别决定走势级别:一个中枢=盘整;两个及以上**同向、同级别**中枢由趋势连接=趋势(上涨/下跌)。

**【可执行判定】**
```
输入:线段序列 seg[](§4)。若暂无线段,用笔近似(降级方案,标注低置信)
1. 取连续三段 seg[i], seg[i+1], seg[i+2]
2. 各段区间 = [min(端点两价), max(端点两价)]
3. ZG = min(三段各自 high); ZD = max(三段各自 low)
4. 若 ZG > ZD → 成中枢,区间 [ZD, ZG],起点=seg[i]起,终点=延伸到的最后一段
5. 延伸:后续段区间与 [ZD,ZG] 仍重叠 → 并入(更新 d_end);不重叠且突破 → 中枢结束
6. 计数中枢个数与方向,用于判断"盘整 vs 趋势"
```

**【对我们引擎的建议】**
- 现有 `_zhongshu` 用**笔的分型价**当高低,取 `zg=min(tops), zd=max(bots)`,思路对但**输入应换成线段**(见 §4)。
- 现有实现有两个问题:
  1. 取 `seg = bi[i:i+4]`(4 个分型点=3 笔)后又 `i += 3`,窗口滑动与"三段"定义对不齐,易漏/重叠。
  2. 中枢合并只按区间重叠,没有区分**同级别延伸**与**升级别扩展**,也不输出中枢**个数/方向**(趋势判定要用)。
- 建议 `_zhongshu` 额外输出:`{zg, zd, level, count_in_trend, direction_of_entry, direction_of_exit}`。

## 6. 三类买卖点

**【定义】(p076/p077)**
以「买点」为例(卖点对称):
- **第一类买点(1B)**:某级别**下跌趋势**的最后一段,出现**趋势背驰**(见 §7)或盘整背驰后的转折,
  即下跌力度衰竭处的最低点。是**趋势方向转折**的起点,级别最高、最安全但最难抓。
- **第二类买点(2B)**:第一类买点出现后,市场**次级别回抽不创新低**的那个低点(第一次回调的底)。
  确认标准:1B 之后一波上涨,再回调,**回调低点 > 1B 低点**(次级别回调背驰)。
- **第三类买点(3B)**:**中枢形成后,一次向上离开中枢,回抽不跌回中枢**的那个低点。
  精确条件:某级别中枢 `[ZD,ZG]`,走势向上离开中枢后回调,**回调低点 > ZG**(不回中枢),该低点即 3B。
  含义=对该中枢的"**次级别走势类型**"的第三类买点,标志新一段上涨(离开中枢向上)。
- 卖点镜像:1S=顶背驰点;2S=1S 后反抽不创新高点;3S=向下离开中枢后反抽**不涨回 ZD**(反抽高点 < ZD)。

**【可执行判定】**
```
1B: 在趋势最后一个中枢后的那一段,MACD/力度背驰成立(§7)→ 取该段极值点
2B: 1B 之后 出现「上涨-回调」,且回调低点 > 1B 低点(且回调是次级别一笔/段)
3B: 已确认中枢 [ZD,ZG];离开中枢方向向上;之后首个回调低点 > ZG → 3B
    (对称:向下离开后反抽高点 < ZD → 3S)
```

**【对我们引擎的建议】**
- **第三类买卖点最易落地**:引擎已有中枢 `[zd,zg]`,只需在中枢确认后,检测**离开中枢的那一笔/段方向**,
  再看**下一个回调/反抽的极值是否守住 ZG/ZD**。输出 `{type:"3B", price, zg, zd, d}`。
- 第一类买卖点依赖背驰(§7),需要先接入 MACD;第二类依赖第一类,优先级排在 1B 之后。

## 7. 背驰(趋势背驰 / 盘整背驰 / MACD 辅助)

**【定义】(p068/p069/p070)**
- 背驰的前提:**必须有趋势**(至少两个同向、同级别中枢由趋势连接)。「没有趋势,没有背驰」。
- **趋势背驰**:趋势中,连接前后两个同向中枢的两段同向走势里,**后一段的力度弱于前一段**。
  典型判据:后一段虽创新高(下跌则创新低),但
  - MACD 红(绿)柱**面积明显缩小**,且/或
  - MACD **DIF/DEA 的高点(低点)背离**(价创新高而指标不创新高),且/或
  - 后一段的价格斜率/成交量能量小于前一段。
- **盘整背驰**:发生在**盘整(单中枢)**内部,中枢两侧的进出力度对比;判据同上但只在一个中枢范围内比较。
- MACD 只是**辅助**:面积=力度,黄白线回拉零轴附近再背离更可靠。趋势背驰对应第一类买卖点。

**【可执行判定】**
```
前置:已识别趋势(同向中枢≥2)与其中"进入中枢段 a"与"离开中枢段 b"(同向)
指标:对每根K算 MACD(12,26,9),取 DIF、DEA、macd_bar = 2*(DIF-DEA)
比较(以底背驰为例,顶对称):
  b 段创新低(收盘/最低价 < a 段极值)
  且 sum(|macd_bar| over b段) < sum(|macd_bar| over a段)   # 绿柱面积缩小
  且 DIF_min(b) > DIF_min(a)                                # 指标不创新低
  → 判定底背驰(趋势背驰)→ 触发 1B 候选
盘整背驰:把 a=中枢第一次下探段、b=中枢最后一次下探段,同法比较
```

**【对我们引擎的建议】**
- 引擎目前**没有 MACD**,也没有"段力度"概念。需新增:
  - 依赖数据:每根K的 `close`(算 MACD),已有 `o/c/h/l/v`,足够。
  - 新函数 `_macd(bars)` 与 `_seg_power(seg, macd_bars)`(段内红/绿柱面积 + DIF 极值)。
- 背驰判定必须**成对比较同向段**,因此需要先有"进入/离开中枢的同向段"的配对逻辑(依赖 §4/§5/§6)。

## 8. 级别递归与结合律(工程近似)

**【定义】(p072/p074)**
- 级别不是固定周期,而是由**中枢级别**递归定义:本级别若干走势类型 → 高一级别一个走势类型。
- 结合律:分解方式在"标准"下唯一,同向相邻可结合;跨级别判断需保证**同级别**才能比较力度/背驰。

**【可执行判定 / 工程近似】**
- 用**周期档位**近似级别:`{5m, 30m, 日线, 周线}`。同一标的多档并行 `analyze`。
- 跨级别对齐:高档中枢的一段 ≈ 低档的一个完整走势类型;用时间区间对齐(高档某段的 `[d_start,d_end]`
  落在低档若干中枢上)。
- **交易含义**:高档定方向(趋势/中枢),低档找买卖点(1B/2B/3B),即"大级别看势、小级别下手"。
