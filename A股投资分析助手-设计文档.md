# A股投资分析助手 — 总体设计文档

> 版本 v1.0 | 2026-07-04 | 作者:John Qiu + Claude
> 定位:个人散户自用的 A股盘前/盘后智能分析系统。**只做分析建议,不做自动下单。**
> ⚠️ 本系统非持牌投顾,所有输出仅供参考,不构成买卖建议。

---

## 1. 项目目标

| 编号 | 目标 | 优先级 |
|---|---|---|
| G1 | 每日自动抓取 A股行情、热点板块、涨停梯队、政策消息 | P0 |
| G2 | 技术面(指标)+ 消息面(新闻政策)综合分析,产出候选股清单 | P0 |
| G3 | 每只荐股附带:逻辑理由 + 同花顺自查路径(技术指标/F10)+ 风险提示 | P0 |
| G4 | 个股加仓/减仓/止损的操作参考策略 | P1 |
| G5 | 导入本人持仓,做针对性诊断 | P1(二期) |
| G6 | 模拟盘验证策略,不用真金白银试错 | P1(二期) |

**明确不做**:自动下单、盘中秒级抢单、涨跌预测承诺。

## 2. 总体架构

```
┌─────────────────────────────────────────────────────┐
│ 调度层  Windows 任务计划 / Cowork 定时任务            │
│         盘前 08:30 · 盘后 15:30 自动触发              │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 数据层  AKShare(主) ⇄ adata(备) 双源互备              │
│         行情/K线/涨停池/板块资金流/龙虎榜/北向/公告    │
│         → 清洗校验 → SQLite 本地库(data/market.db)   │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌──────────────────────────────┬──────────────────────┐
│ 分析层·技术面(纯代码计算)     │ 分析层·消息面(AI)     │
│ MA/MACD/RSI/量比/乖离率/换手  │ 联网搜索当日政策热点   │
│ → 6维度100分评分              │ → AI 归纳板块催化剂    │
└──────────────┬───────────────┴──────────┬───────────┘
               ▼                          ▼
┌─────────────────────────────────────────────────────┐
│ 决策层  AI 综合研判(技术面60% + 消息面30% + 大盘10%) │
│         硬性规则过滤 → 强制反方理由 → 止损/目标价     │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 输出层  每日报告(Markdown)→ 项目文件夹 reports/       │
│         格式:板块→逻辑→个股→同花顺自查→风险提示      │
└─────────────────────────────────────────────────────┘

侧翼模块(二期):持仓诊断(CSV导入→easytrader→miniQMT)
              模拟盘(paper trading,虚拟账户验证策略)
```

*名词解释:双源互备=同一数据从两个渠道取,一个挂了用另一个;SQLite=存在本地的轻量数据库文件,无需装服务器。*

## 3. 模块详细设计

### 3.1 数据层

**数据源与接口清单(全免费):**

| 数据 | 主源(AKShare 接口) | 备源 | 更新频率 |
|---|---|---|---|
| 实时行情快照 | `stock_zh_a_spot_em` | adata | 盘中约3秒延迟 |
| 日K线(前复权) | `stock_zh_a_hist` | adata | 每日 |
| 涨停池/炸板池 | `stock_zt_pool_em` / `stock_zt_pool_zbgc_em` | — | 盘后 |
| 板块行情+资金流 | `stock_board_industry_name_em` + 资金流接口 | adata | 盘中/盘后 |
| 龙虎榜 | `stock_lhb_detail_em` | — | 盘后 |
| 个股公告/新闻 | `stock_notice_report` / 新闻接口 | AI 联网搜索 | 每日 |
| 财务指标(F10类) | `stock_financial_abstract` 等 | — | 季度 |

**设计要点:**

- 所有接口调用封装在 `data_fetcher.py`,统一加:超时重试(3次)、限速(防封IP)、双源降级、数据落库
- 落库后做**三项校验**:非空校验、价格区间合理性校验(涨跌幅超±20%非ST报警)、时间戳新鲜度校验(数据早于当日则拒用并标记)
- SQLite 三张核心表:`daily_kline`(日线)、`snapshot`(快照)、`events`(涨停/龙虎榜/公告)

### 3.2 分析层 — 技术面引擎

**原则:所有数字由 Python 算,AI 只解读不计算**(大模型心算易错,这是正确性的第一道防线)。

指标集(pandas-ta 计算):MA5/10/20/60、MACD(12/26/9)、RSI(6/12/24)、量比、换手率、乖离率、20日支撑压力位。

**评分系统**(借鉴 Stock-Analysis-Skill 的 100 分 6 维度制,可直接复用其代码):

| 维度 | 满分 | 说明 |
|---|---|---|
| 趋势(均线排列) | 30 | 多头排列高分,空头排列 0 分 |
| 乖离率 | 20 | 贴近 MA5 高分,追高(>5%)低分 |
| MACD | 15 | 零轴上金叉最高,死叉 0 分 |
| 量能 | 15 | 缩量回调最高,放量下跌 0 分 |
| RSI | 10 | 超卖高分,超买 0 分 |
| 支撑 | 10 | 双均线支撑高分 |

信号映射:≥75 强烈关注 / ≥60 关注 / ≥45 持有 / ≥30 观望 / <30 回避。

### 3.3 分析层 — 消息面引擎

每日流程:联网搜索「当日 A股 热点板块 政策」→ 抓取财联社/东财要闻 → AI 按板块归纳:催化剂是什么、是短期情绪还是中期逻辑、受益个股有哪些 → 与技术面评分交叉:**只有"消息面有催化 + 技术面评分≥60"的板块才进入荐股池**。

### 3.4 决策层

**硬性规则(代码强制,AI 不可绕过):**

1. RSI > 80 或 MA5 乖离率 > 5% → 禁止给出"关注/买入"信号(不追高)
2. ST股、上市不满 60 日的次新股 → 直接过滤(按你的风格约束配置在 config)
3. 每只荐股**必须**输出:止损价(基于 MA20 或近期低点)、目标价(基于压力位)、失效条件
4. 每只荐股**必须**输出至少 2 条反方理由(防 AI 报喜不报忧)
5. 单日荐股池上限 5 只(防信息过载)

**输出格式(固定模板):**

```
【板块】→【逻辑:为什么涨】→【代表个股+评分】
→【同花顺自查路径:如何看该指标/F10哪一页验证】
→【操作参考:入场区间/止损/目标】→【风险提示】
```

同花顺自查指引示例:"MACD 金叉可在同花顺个股页按 VA 键切换指标验证;商誉风险看 F10→财务分析→资产负债表;股东人数变化看 F10→股东研究"。

### 3.5 持仓模块(二期,三步走)

| 阶段 | 方案 | 门槛 |
|---|---|---|
| 2a | 同花顺 App/PC 导出持仓 CSV → 放入 `portfolio/` 文件夹 → 系统自动逐股诊断 | 零门槛,先用这个 |
| 2b | easytrader 对接同花顺客户端自动读持仓 | 需 PC 常开客户端,略折腾 |
| 2c | 券商开通 miniQMT(华泰/国信等,门槛约10万资金)→ 程序直连查持仓 | 最正规,留到三期 |

持仓诊断输出:每只持股的技术面评分变化、所属板块消息面状态、加/减/止损建议+理由。

## 4. 开源 Skill 融合方案(提高生产力的关键)

调研了 GitHub 上现成的 Claude Skills,按契合度排序:

### 4.1 直接安装使用 ⭐⭐⭐

**[a-share-skill](https://github.com/shouldnotappearcalm/a-share-skill)**(171 star,专为A股)
- `a-share-data`:实时行情/历史K线/技术指标/事件/板块/北向资金查询 → **直接覆盖本项目整个数据层+技术面引擎**
- `a-share-paper-trading`:模拟账户/模拟买卖/回测 → **直接覆盖二期模拟盘模块**
- 附赠策略:`macd-second-golden-cross`(MACD二次金叉)、`macd-trend-resonance-stock-picker`(均线+MACD共振选股)、`tuige-shortline-trading`(短线触发/失效/仓位决策)
- 融合方式:克隆仓库,把 skill 文件夹放入本项目,作为基础设施

### 4.2 借鉴融合核心逻辑 ⭐⭐

**[Stock-Analysis-Skill](https://github.com/liusai0820/Stock-Analysis-Skill)**(55 star)
- 精华在:100分6维度评分系统、"严进策略"硬性规则、数据源分级降级链(Tushare→efinance→akshare→yfinance)、决策看板输出模板
- 融合方式:不整体安装,把它的 `stock_data_fetcher.py`(约400行,含降级逻辑)和评分规则移植进本项目决策层

### 4.3 参考架构 ⭐

- **[daily_stock_analysis](https://github.com/ZhuLinsen/daily_stock_analysis)**:定时运行+多源新闻+自动推送的整体架构参考(上面 Stock-Analysis-Skill 的灵感来源)
- **[finance-quant-skills](https://github.com/lzwme/finance-quant-skills)**:A股量化策略开发类 skill 库,三期做回测时再取用
- **[UZI-Skill](https://github.com/wbh604/UZI-Skill)**:66位投资大佬风格模拟、180条量化规则,娱乐性强,可选装做"第二意见"

### 4.4 融合后的自建清单(只剩三块要自己写)

1. `config.yaml`:你的风格约束(不碰ST/次新、仓位偏好、最大回撤容忍度)
2. `daily_report.py`:按第 3.4 节固定模板生成每日报告 + 同花顺自查指引知识库
3. 调度脚本:Windows 任务计划注册 + 失败重试

**结论:融合开源 Skill 后,一期工作量从约 3 周压缩到约 1 周。**

## 6. 正确性保障机制

| # | 机制 | 防什么 |
|---|---|---|
| 1 | AI 零计算原则:指标/评分全由代码产出 | 大模型算数出错 |
| 2 | 双源交叉校验:关键价格两源比对,偏差>0.5%标警告 | 单一爬虫源出错 |
| 3 | 数据新鲜度门禁:时间戳过期即拒绝输出结论 | 拿昨天数据当今天 |
| 4 | 硬性规则引擎:追高/超买/ST 等由代码一票否决 | AI 被行情叙事带偏 |
| 5 | 强制反方理由+失效条件 | AI 只说多头故事 |
| 6 | 模拟盘先行:新策略先跑模拟账户≥1个月再参考 | 未经验证的策略 |
| 7 | 回测验证(三期):历史数据检验胜率/最大回撤 | 幸存者偏差 |
| 8 | 人在回路:系统只荐不买,下单永远人工 | 自动化事故 |

## 8. 风险与合规

- **市场风险**:任何技术面/消息面分析都不能预测涨跌,评分高≠会涨。
- **数据风险**:免费源为爬虫机制,有延迟且可能临时失效,已用双源+校验缓解,但不适合盘中秒级决策。
- **第三方代码风险**:开源 Skill 引入前需通读其代码,确认无上传账户信息等行为;模拟盘数据仅存本地。
- **合规声明**:本系统为个人研究工具,输出不构成投资建议;使用 miniQMT 等券商接口需遵守券商协议;不得将输出用于荐股类商业用途(涉《证券法》持牌要求)。
