# A股投资分析助手 — 使用说明

**当前版本:v3.6** · 个人研究工具 · Python + AKShare/DangInvest + ECharts

> ⚠️ 本工具仅供个人研究参考,不构成投资建议。数据来自免费接口,可能有延迟或临时失效。

## 架构总览

```
                    ┌─────────────────── 数据层(韧性) ───────────────────┐
   免费数据源  ───▶ │ fetcher(双源降级+缓存+限速+Chrome指纹) · danginvest │
 东财/新浪/同花顺   │ 三层兜底:当日缓存 → 实时(主源→备源)→ 陈旧缓存兜底   │
 /DangInvest        └───────────────────────┬─────────────────────────────┘
                                             ▼
        ┌──────────── 分析层(纯代码计算,AI只解读) ────────────┐
        │ indicators(MA/MACD/RSI/量比/乖离) · scoring(6维100分) │
        │ chips(筹码CYQ) · chan(分型/笔/线段/中枢/买卖点/背离)   │
        │ candle_patterns(70形态库检测) · knowledge/kb(领域知识)│
        └───────────────────────┬───────────────────────────────┘
                                 ▼
     ┌───────── 入口 ─────────┐        ┌──────── 输出/技能 ────────┐
     │ daily_report 热点板块池 │        │ render-html 交互图文HTML  │
     │ global_scan 全市场池    │  ──▶   │ (下拉切股+4Tab+筹码+资金) │
     │ analyze 点名分析        │        │ backtest-review 复盘命中率│
     └────────────────────────┘        │ trading-memory 操盘台账   │
                                        │ news-analysis 消息面      │
                                        └───────────────────────────┘
```

**核心原则**:①AI 零计算(指标/评分/筹码/缠论全由代码算,AI 只解读);②数据韧性(双源+缓存,单源挂了不影响);③知识驱动(缠论/K线形态蒸馏成 `knowledge/kb/` 文件供代码调用);④强制反方理由 + 硬规则否决(不报喜不报忧)。

## 数据源与韧性(重要)

免费接口会限流/临时失效,本工具用**三层兜底**保证「永远跑得完」。当前板块主源已默认
`danginvest`(避开东财限流,体验更稳):

| 数据 | 主源 | 备源 | 再兜底 |
|---|---|---|---|
| 三大指数 | 新浪 | — | 缓存 |
| 板块排名 | **DangInvest** | 东财 push2 | 缓存 |
| 板块成分股 | **DangInvest** | 东财 push2 | 缓存 |
| 个股资金流 | 东财 push2his | — | 缓存/优雅缺省 |
| 个股日K | 东财 push2his | **新浪** | 缓存 |
| 涨停池 | 东财 | — | 缓存 |

- **缓存**:当天已取的数据直接复用(省请求、避限流);实时失败则回退最近一次缓存并标记陈旧,由新鲜度门禁(默认>4天拒用)把关
- **双源自洽**:板块排名走到哪个源,其成分股就用同源接口,命名不会对不上
- 即使东财**完全挂掉**,系统也能靠 DangInvest + 新浪出一份完整报告

## 常见问题:东财接口连不通 / ProxyError / Connection closed

按可能性从高到低:

1. **系统代理**:`fetcher.py` 已强制 `NO_PROXY` 直连,一般无需理会;若仍报 ProxyError,关掉系统代理再跑。
2. **东财限流(最常见)**:短时间请求过多,东财会临时断连该 IP(几十分钟量级)。表现为 `Connection closed abruptly`。应对:
   - **等**:静置一段时间会自动恢复,别反复重试(重试会续上冷却)
   - **换 IP**:重启光猫/路由器重新拨号,或临时用手机热点(封禁认 IP,换了立即生效)
   - 已默认 `prefer_source: danginvest`,板块数据**彻底不碰东财 clist**;板块图表里「领涨股」列会留空(用东财主源才有)
3. **资金流/K线偶发缺失**:走东财 push2his,限流时资金流面板自动省略、K线自动切新浪,报告照常产出。
4. 正常日用一天只跑一两次,基本不会触发限流。

## 修改配置(config.yaml)

- `prefer_source`:板块主源 `danginvest`(默认,避开东财限流) / `eastmoney`(全,含领涨股)。失败自动互备
- `danginvest_mode`:板块维度 `industry`申万大类 / `sub`同花顺细分 / `concept`概念
- `chart_bars`:图表版 K线展示根数(当前 120)
- `request_interval_sec`:接口间隔(当前 1.0,会 +0~50% 抖动)
- `max_retries`:重试次数(当前 2,压低以免限流时续封禁)
- `max_drawdown_pct`:止损容忍(当前 8) · `min_score`:入池门槛(当前 60)
- `exclude_st` / `exclude_sub_new` / `exclude_bse`:ST/次新/北交所开关
- `top_sectors` / `stocks_per_sector`:扫描广度(当前 3×5)

## 报告怎么读

- **评分 ≥75 且多头排列** = 重点关注;**60~74** = 关注
- 触发硬规则(RSI6>80 超买 / MA5乖离>5% 追高)的股**直接不进池**——这是保护
- 每只股必附:止损价(MA20 与 -8% 取高者)、支撑/压力、反方理由、同花顺自查路径
- **荐股池为空是正常输出**:当天没有低风险入场点,空仓也是操作

## 七个自建 Skill

- **操盘记忆**(`skills/trading-memory/`):对我说「我买了/卖了/复盘/看我的操盘记录」,
  我会记台账并统计胜率、盈亏比、持仓天数、**止损纪律**(扛单会被标出)。
- **作战方案**(`skills/action-plan/`):对我说「作战方案/今日怎么操作/加减仓建议」,把**持仓+荐股池**跨股合成为带价位的组合动作——持仓端(清仓/减仓/持有/逢低加仓+触发价)、荐股端(建仓/逢低吸/逢高突破+计划价/止损/目标/盈亏比)、组合姿态(regime+情绪→总仓位、集中度预警)、换股(卖弱买强)。`--focus 持仓` 或 `--focus 代码/名称` 可收窄。`run_workflow` 会自动生成并渲染成盘前报告顶部「🎯今日作战方案」卡片。计划价位同时是盘中监控的告警阈值来源。
- **盘中监控**(`skills/intraday-watch/`):对我说「盯盘/盘中监控/到价提醒」,**须在你本机、交易时段跑** `python scripts/watch.py`。读当日作战方案的计划价位,腾讯批量快照 15~30 秒轮询持仓+荐股,触及 止损/介入/突破/减仓/涨停炸板 时**桌面弹窗**(win11toast 最佳,无则 MessageBox 兜底)+ 告警台账 + **自刷新分时页**(双击开,分时线+计划价位标线)。只绑事先算好的计划价位、每类每日限次防轰炸;只读只提醒,不下单。`--once` 可休市冒烟测。
- **持仓诊断**(`skills/portfolio-diagnosis/`):从券商/同花顺 APP **导出持仓文件**(代码/名称/数量/成本价那张表)
  放进 `portfolio/`,对我说「诊断持仓」。逐只结合**成本价**给 持有/减仓/止损/观察 的规则化诊断(距结构止损多远、
  是否破位、浮亏是否超容忍),并出**组合体检**(仓位/集中度/主力资金流出家数)。🔒 只读持仓、**不需要也绝不接收账户密码、不做实盘下单**;`portfolio/` 已在 .gitignore,不会被提交。
- **渲染 HTML**(`skills/render-html/`):
  - `md_to_html.py`:任意 Markdown → 富文本网页(表格/卡片/涨跌红绿/信号徽标)
  - `report_to_html.py`:日报/点名/全局 `.json` 侧车 → **交互图文版**(深色面板,下拉切股 + 4个Tab:
    **K线** / **缠论**(分型/笔/中枢/1B/3B买卖点/背驰) / **形态**(支撑压力/颈线/双顶双底) / **K线形态**(70种组合标注) + 筹码分布 + 资金流)
  - 图表基于 ECharts(CDN 加载,查看需联网);其余纯标准库,无需装依赖
- **盘后复盘**(`skills/backtest-review/`):对我说「复盘今天的荐股/本周表现/历史命中率」,
  我用荐股的目标/止损对照真实走向判定命中,累积长期成功率(先触目标=胜、先触止损=败)。
- **消息面**(`skills/news-analysis/`):对我说「今日热点/消息面」,我联网搜政策/热点/公告,
  按「板块→催化剂→情绪or逻辑→受益股」归纳并与技术面交叉,注入日报第三章。
